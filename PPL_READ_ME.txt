Скрипт использует стандартный python SDK, предоставляемый AliExpress. Некоторые базовые файлы, в частности base.py, были переписаны под работу с Python 3.8.
Также в base.py внесены дополнительные небольшие изменения (костыли) с целью упростить работу собственного кода.
-----------------------------------------------
Список собственных файлов:
-----------------------------------------------
1) config_data.py - содержит класс ConfigData, который хранит в себе все необходимые настройки для подключения к сервисам AliExpress и 1С
2) ae_general_product_updater.py - содержит класс AEGeneralProductUpdater, ответственный за основной ход операции по обновлению остатков и цен на AE, а также за сбор данных о текущих товарах, находящихся в продаже на AliExpress. Операция запускается из метода данного класса launchGeneralProductUpdate().
3) ae_product_batch_update.py - содержит класс AEProductBatchUpdater, который отвечает исключительно за операции по обновлению остатков и цен. Методы данного класса вызываются из метода launchGeneralProductUpdate() класса AEGeneralProductUpdater.
4) ae_logger.py - содержит класс AELogger, отвечающий за запись ошибок, возникших в ходе обновления операций. Создает логи в папке ae_error_logs, по одному файлу логов на дату. Все записи в логах содержат временные штампы чтобы лучше понимать, когда, при каком запуске скрипта произошла ошибка.
5) update_launcher.py - файл с точкой входа, который запускает операцию обновления данных.
-----------------------------------------------
Порядок работы:
-----------------------------------------------
1) Открыть проект в PyCharm
2) Открыть update_launcher.py и запустить скрипт
3) Если файл запуска отсуствует, то создать новый файл с точкой входа (main), и вызвать в ней метода класса AEGeneralProductUpdater - launchGeneralProductUpdate().
-----------------------------------------------
ВАЖНО:
----------------------------------------------- 
1) Все данные в config_data.py на текущий момент - это данные компании "***". Если скрипт будет переиспользован, то необходимо будет сделать соответствующие изменения.
2) API AliExpress использует OAuth 2.0. Данный скрипт не имеет кода для прохождения авторизации т.к. изначельно планировался для запуска вручную. Вместо этого, в config_data.py хранится текущий рабочий токен доступа, также сгенерированный вручную. Токены доступа к ALieExpress API действуют в течении года для собственных разработчиков (мы используем аккаунт собственного разработчика компании "***"), и их надо регулярно менять на новые.
